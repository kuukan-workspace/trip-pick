# TripPick MVP Plan

## 서비스명
TripPick

## 한 줄 설명
인기 여행 도시를 기준으로 항공, 숙소, 음식 정보를 한 번에 탐색할 수 있는 여행 정보 큐레이션 웹서비스

## 1차 대상 국가
- 일본

## 1차 대상 도시
- 도쿄
- 오사카
- 후쿠오카

## 주요 사용자 흐름
1. 메인 페이지에서 일본 도시 선택
2. 도시 상세 페이지 진입
3. 항공 / 숙소 / 음식 섹션 탐색
4. 음식 카드에서 상세 정보 확인
5. 향후 외부 서비스 링크로 이동

## 핵심 섹션
### 항공
- 예시 최저가
- 항공사
- 노선
- 비고(실시간 아님)

### 숙소
- 호텔
- 료칸
- 로컬숙소

### 음식
- 라멘
- 스시
- 이자카야
- 카페
- 도시별 특화 카테고리 가능

## 기술 목표
- React/Next 기반 라우팅 구조 익히기
- 정적 데이터 관리 구조 만들기
- 카드 UI, 섹션 UI, 상세 정보 UI 만들기
- 나중에 API 연동 가능한 확장 구조 잡기

## 구현 우선순위
### P1
- 메인 페이지
- 도시 상세 페이지
- 더미 데이터 연결

### P2
- 카테고리 필터
- 음식 상세 카드 개선
- 반응형 UI 개선

### P3
- 지도 링크
- 외부 예약 링크
- API 연동 준비
